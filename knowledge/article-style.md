# 智谷網絡 — 文章風格指南
資料來源：kvalley.biz 文章分析（24362 + CEO觀點分類）

## 01 整體語調

**核心定位：** 像有實戰經驗的顧問在說真心話，輕鬆但有深度

| 維度 | 描述 |
|------|------|
| 語氣 | 親近對話式，不是教科書式 |
| 立場 | 陪伴者、夥伴，不是上對下教導者 |
| 深度 | 有觀點、有框架，不只說表面 |
| 實用性 | 每篇都要有「今天能做的第一步」 |

---

## 02 開頭風格

- 用「我們」或第一人稱，展現誠懇態度
- 從真實痛點切入，不從定義或理論切入
- 前兩句要讓讀者覺得「這說的就是我的問題」
- 範例：「陪著台灣企業在人才發展這條路上跑了 30 年⋯⋯」

**好的開頭公式：**
> 場景 / 現象 → 點出矛盾 → 我們的立場

## 03 常用語言習慣

**口語化詞彙（可適度使用）：**
- 「欸」「坦白說」「其實啦」
- 「說真的」「你有沒有發現」
- 「先說結論」「說個真實的情況」

**品牌專屬語言：**
- 用「夥伴」不用「客戶」
- 用「蛻變」不用「改變」
- 用「落地」不用「實施」
- 用「推不動」直接引用讀者語言（增加共鳴）

**避免：**
- IT 術語（API、LLM、token、prompt engineering）
- 生硬的 ROI 百分比（沒有情境脈絡的數字）
- 模糊詞：「有助於」「可能」「也許」
- 空泛標語：「AI 是未來」「數位轉型刻不容緩」
- **AI 式話術句型（Chloe 2026-04-17 指示）** — 已經被大眾貼上「AI 味」標籤，讀者會秒辨識：
  - ❌ 「這不是 X，而是 Y」（最明顯的 AI 味）
  - ❌ 「與其說是 X，不如說是 Y」
  - ❌ 「它不僅是 X，更是 Y」
  - ❌ 排比三連「是⋯、是⋯、是⋯」當結尾重擊
  - 替代寫法:直接說 Y，或用「X 聽起來對，但真正的問題在 Y」「表面上 X，底下其實 Y」這類更自然的結構
- **em-dash（——）全面禁用 ★（Chloe 2026-04-29 升級為零容忍，原 2026-04-22 上限規則作廢）** — 不論 H1／H2／內文／CTA，**一律不准出現 `——` `—`**。LLM 慣性產出，讀者秒辨識成 AI 文。
  - 替代寫法：
    - 句號斷兩句：「這是 AI 的問題——不只是技術」→「這是 AI 的問題。但不只是技術。」
    - 逗號或冒號：「同一個訊號——從不同產業發出來」→「同一個訊號，從不同產業發出來」
    - 冒號帶定義：「真正要解的是另一組問題——CEO 預告的是⋯」→「真正要解的是另一組問題：CEO 預告的是⋯」
    - 括號補充：「這個方法——快、準、便宜——適合急用」→「這個方法（快、準、便宜）適合急用」
  - Persona 審核必做：寫完後 `grep -nE '——|—' <article>.md` 命中即退稿，不論幾個

## 04 文章結構

### 標準長度：2,500–3,500 字

### 推薦架構（以珊珊為目標讀者）：
1. **開場場景**（100–200 字）— 讀者當下正在面對的情境
2. **點出矛盾**（150 字）— 為什麼現有做法不夠？
3. **核心觀點 / 框架**（600–800 字）— 3–5 個要點，每點有小標題
4. **真實案例或數據**（400–500 字）— 同類企業 / HR 的具體經驗
5. **今天能做的第一步**（200–300 字）— 清晰、不嚇人的行動指引
6. **結尾邀請**（100 字）— 開放式問句，邀請讀者回應或分享

### 推薦架構（以數位轉型推手為目標讀者）：
1. **戰略視角開場**（150 字）— 點出更大格局的問題
2. **為什麼現在是關鍵時刻**（200 字）— 外部壓力 + 機會窗口
3. **降維打擊框架**（500–700 字）— 戰略性思維，不只說操作
4. **說服老闆的語言彈藥**（400 字）— 可複製的數字和句子
5. **同產業 ROI 案例**（400 字）— 越具體越好
6. **今天的第一步**（200 字）— 組織層級的起始動作
7. **行動邀請**（100 字）

## 05 格式規範

| 元素 | 規範 |
|------|------|
| 標題層級 | `## ` → WP H1 灰藍背景大字（章節分隔）；`### ` → WP H2 medium 字級（子標）；H1 文章標題只有開頭一個 |
| H2 編號規則 | How-to 教學用 `Step 1～N`；論述／拆解／問題驅動型用「一、二、三、」；敘事型不編號用敘事文字 H2 |
| **破折號全面禁忌（2026-04-29 升級）★** | 全文（H1／H2／內文／CTA）**一律禁破折號 `—` `——`**。H1 改用冒號 `：`；H2 改用 `，` `：` `｜`；內文用句號／逗號／冒號／括號（替代寫法見第 03 節）。Persona 審核 `grep -nE '——\|—'` 命中即退稿 |
| **Blockquote 禁忌（2026-04-29）** | 文章內**全面禁用 markdown `>` blockquote 語法**（含引語錄、強調段、警示框等任何用途）；引用語錄改用內文 `「⋯⋯」` 或自成一段的粗體強調，需要視覺斷裂感用粗體＋空行 |
| **水平分隔線禁忌（2026-04-29 新增）★** | 文章內**禁用 markdown `---` 三連減號**（WP 會渲染成莫名橫線，破壞版面）；要分段就空行＋下一個 H2/H3，不要靠 `---` 製造視覺斷點 |
| 表格 | 適合呈現比較類、列舉類資料（3欄以內最易讀）|
| 條列清單 | 每條盡量 15–25 字，不超過 5 條一組 |
| 粗體（純黑） | 結構性次標題、列表項目（事實 1、訊號 1、Step、第一層、做法一、坑 1 等）用 `<strong>...</strong>` 寫死純黑；publish_to_wp 不會上色 |
| 粗體（橘色 #ff6900） | `**xxx**` 短關鍵字（不以句號結尾）→ 自動橘色；**一篇 5-6 個以內**，只給「重點中的重點」。寧可一段都不標 |
| 粗體（粉紅 #f78da7） | `**xxx。**` 整段語句（句號結尾）→ 自動粉紅；只給「整段都該被讀者抓到」的句子，一篇 2-4 個 |
| 推薦文章 | frontmatter 寫 `**推薦文章**：<id1>, <id2>`，按上架順序前兩篇（這篇上架時、之前最新的兩篇）；不寫的話 publish_to_wp fallback 抓 WP 最新兩篇 |
| 圖片 | 至少一張封面圖；AI 生成圖（與品牌調性一致）|
| 標籤 / Tag | publish_to_wp 自動用主關鍵字做 tag，不用另外列 |
| CTA | 用 markdown link `[xxx](url)`；URL 不要露在文字裡。一句話寫精準，不寫 generic「歡迎找我們聊聊」 |
| 內部連結 | **直接 inline 寫進內文**至少 5 個，混搭服務頁＋相關文章；**禁止**在文末列「內部連結建議」清單（會被 publish_to_wp 推上去佔版面） |

## 06 SEO 基本規範

- **標題**：含核心關鍵字，50–65 字元以內
- **Meta Description**：120–155 字元，含行動誘因
- **H2 標題**：自然含入次要關鍵字（不要硬塞）
- **內部連結**：每篇文章至少 2–3 個內部連結（參考 sitemap.md）
- **段落長度**：每段落不超過 4 行，方便行動裝置閱讀

## 08 寫作 Checklist（Bot 2 SEO 寫手必核）

寫完初稿後，確認以下項目：

- [ ] 開頭前兩句能讓目標讀者說「這說的是我」？
- [ ] 有沒有使用真實語言（引用問卷語言或具體場景）？
- [ ] 核心觀點是否有 3–5 個清楚的小標題？
- [ ] 有沒有表格或條列，讓資訊容易掃描？
- [ ] 「今天能做的第一步」是否具體、不嚇人？
- [ ] 是否有 2–3 個內部連結？（參考 sitemap.md）
- [ ] 是否避開了 IT 術語和空泛標語？
- [ ] SEO 標題是否 65 字元以內，含關鍵字？
- [ ] Meta Description 是否 155 字元以內？
