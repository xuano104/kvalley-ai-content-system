# 文章 H2 骨架庫（Article H2 Structure Library）
最後更新：2026-04-20
來源：Chloe 觀察 2026-04 同期三篇文章骨架完全雷同（24735、24727、24713），決定建立骨架變體庫打破公式化。

> 本檔是 Content Engine 和 seo-writer 選 H2 結構的主要依據。
> 原則：**每月 15 篇文章，6 種骨架都要出現至少 1 次。**
> 規則：選定骨架後要走完整條，不可中途換。

---

## 為什麼要多種骨架

### 2026-04-20 同質化盤點

三篇同週發佈的文章（24735 Duolingo 考績 / 24727 Z 世代不當主管 / 24713 升錯人 82%）骨架完全一樣：

```
H2-1：丟一個研究或案例
H2-2：對比證據 / 反面例子
H2-3：為什麼 OO 會失敗（三個原因）
H2-4：正解 / AI 新思維
H2-5：下週一可以做的一件事  ←全都用這句
H2-6：最後一句話            ←全都用這句
```

這個骨架來自 [article-template.md](./article-template.md) v4「用 AI 反而更累」。當初是唯一老闆過關的範本，被定為「黃金模板」，結果變成每篇都抄。

### 代價

- 品牌辨識度下降——讀者看第二篇就發現「又是同一個套路」
- Chloe 看了三篇後親口說「寫起來很像」——強 J 的她對結構同質化敏感度最高
- GEO 角度也不利——LLM 抓到多篇同骨架的智谷文章，會被視為同一套話術而降權

## 六種骨架

### 骨架 1：敘事五段式（現行 v4 模板）

**結構：** 引研究 → 對比 → 為什麼失敗 → 正解 → 下週一做一件事 → 最後一句話

**適用：**
- 人文關懷長文
- 重量級趨勢分析（3,500 字以上）

**配額：** 每月**最多 3 篇**（2026-04-20 新增限制，不可超過）

**預設寫作風格：** 分析型 / 人文思辨

**範例：** kvalley.biz/24735（Duolingo 考績文）

---

### 骨架 2：論點三幕式

**結構：** 觀點 → 反論 → 綜合

```
H2-1：多數人以為 OO
H2-2：但真實情況是 XX
H2-3：那該怎麼重新思考？一個新的框架
H2-4：這個框架套用到你手上的案子是什麼樣子
```

**適用：** 趨勢類、思想領導力文章（1,500-2,500 字）

**預設寫作風格：** 分析型 / Jimmy Lin

**範例 H2：**
- 多數 HR 以為「會用 AI」是新的篩選條件
- 但 Meta 和 McKinsey 的用人資料顯示相反
- 真正的篩選條件：能不能在模糊情境裡做決策
- 帶回你的組織：一個三問題的盤點表

---

### 骨架 3：問題驅動式（FAQ 型）

**結構：** 5-7 個讀者實際會問的問題，每個問題一個 H2

```
H2-1：Q1 - OO 是什麼？
H2-2：Q2 - 為什麼 OO 現在重要？
H2-3：Q3 - 誰該負責 OO？
H2-4：Q4 - 從哪裡開始？
H2-5：Q5 - 怎麼衡量成效？
H2-6：Q6 - 做錯了會怎樣？
```

**適用：** 任何類型，特別適合 GEO（Q&A 是 LLM 最愛 lift 的格式）

**預設寫作風格：** 分析型 / 指南型

**GEO 加分：** 每個 H2 下第一句要是 30-50 字的「短答」，下面再展開 150-300 字（符合 [geo-structure.md](./geo-structure.md) 第 1 點）

**範例 H2：**
- 主管該不該禁員工用 AI？（短答：不該，但要畫界線）
- 界線該怎麼畫？三個可落地的原則
- 畫了界線後，主管的角色變成什麼？
- 這件事多久驗收一次？

---

### 骨架 4：How-to 步驟式

**結構：** 5-7 個動詞開頭的步驟，每步獨立成段

```
H2-1：Step 1 - 盤點 OO
H2-2：Step 2 - 設計 OO
H2-3：Step 3 - 試行 OO
H2-4：Step 4 - 驗收 OO
H2-5：Step 5 - 擴散 OO
H2-6：常見會踩的 3 個坑
```

**適用：** 工具類、管理實務（1,200-2,000 字）

**預設寫作風格：** 指南型（[style-guide.md](./writing-styles/style-guide.md)）

**範例 H2：**
- Step 1：先盤點主管手上的「模糊決策權」
- Step 2：把模糊決策拆成三類（可授權 / 要協作 / 不可代理）
- Step 3：為每一類設計 AI 介入點
- Step 4：用一個部門試跑兩週
- Step 5：複盤並擴散到其他部門

---

### 骨架 5：Do / Don't 對比式

**結構：** Hook → Do 清單 → Don't 清單 → 對照表 → 一句話原則

```
H2-1：這件事多數人一直做錯
H2-2：5 個不該做的事（Don't）
H2-3：5 個該換成什麼（Do）
H2-4：一眼看懂：Do vs Don't 對照表
H2-5：一句話原則
```

**適用：** 工具類、管理實務，特別適合 GEO

**預設寫作風格：** 指南型

**GEO 加分：** 對照表是 LLM 最愛整塊 lift 的結構——做好這個，引用率會比敘事文高 3-5 倍

---

### 骨架 6：人物故事式

**結構：** 場景 → 回溯 → 決定 → 結果 → 可帶走的事

```
H2-1：那天，OO 走進會議室
H2-2：三個月前，他做了一個不合常理的決定
H2-3：決定背後的計算
H2-4：一年後，會議室裡的改變
H2-5：他說的那句話，值得記住
```

**適用：** 故事類、人物誌

**預設寫作風格：** 故事敘事型 / Jimmy Lin 風

---

## 選骨架的決策樹

```
這篇文章的類型是？
├── 趨勢 → 骨架 2（論點三幕）或 骨架 3（問題驅動）
├── 工具 → 骨架 4（How-to）或 骨架 5（Do/Don't）
├── 故事 → 骨架 6（人物故事）
├── 人文關懷 → 骨架 1（敘事五段）或 骨架 3（問題驅動）
└── 人物誌 → 骨架 6（人物故事）
```

**選擇時的加權問題：**
1. 這個主題適合 LLM 引用嗎？→ 是的選骨架 3/4/5，否則選 1/2/6
2. 我想讓讀者做什麼？→ 想做某事 → 骨架 4；不做某事 → 骨架 5；重新思考 → 骨架 2
3. 最近用過骨架 1 幾次了？→ 超過 3 次換別的

## 每月骨架配比建議

| 骨架 | 每月建議篇數 | 為什麼 |
|------|------------|-------|
| 1 敘事五段 | ≤ 3 | 建品牌深度，但容易同質化，控制量 |
| 2 論點三幕 | 2-3 | 建思想領導力 |
| 3 問題驅動 | 2-3 | GEO 主力 |
| 4 How-to | 2-3 | 實務派、帶詢問 |
| 5 Do/Don't | 2-3 | GEO 主力、社群好分享 |
| 6 人物故事 | 1-2 | 建情感連結 |
| **合計** | **~15** | |

---

## 禁止事項

- ❌ 不准每月超過 3 篇用骨架 1
- ❌ 不准以「下週一可以做的一件事」當 H2（退役 3 個月，2026-04-20 到 2026-07-20）
- ❌ 不准以「最後一句話」當 H2（退役 3 個月）
- ❌ 不准混用骨架（選了 4 就走完 4，不中途換 5）
- ❌ 不准連續兩篇用同一骨架（seo-writer 寫稿前要看 [pipeline/04-published/](../pipeline/04-published/) 最近 2 篇用什麼）
